## 三、业务逻辑说明

### 3.1 整体数据流

```
Rust 模块（成员 A）
    │  ParsedData（gRPC）
    ▼
DocumentAuditorServiceImpl.auditRules()
    │
    ├─► FormattingAuditor.checkFormatting()
    │       │  输入：ParsedData.sections
    │       │  规则：formatting.drl（20条）
    │       └─► 输出：List<Issue>（排版问题）
    │
    ├─► ReferenceChecker.checkReferences()
    │       │  输入：ParsedData.references + sections
    │       │  规则：reference.drl（26条）
    │       └─► 输出：List<Issue>（引用问题）
    │
    └─► IntegrityScanner.scanIntegrity()
            │  输入：ParsedData（全量）
            │  规则：integrity.drl（11条）
            └─► 输出：List<Issue>（章节完整性问题）
                    │
                    ▼
            AuditResponse（合并所有 Issue + 计算 score_impact）
                    │
                    ▼
        Go 网关（成员 D）→ Python 大模型（成员 C）
```

### 3.2 输入数据格式（ParsedData）

Java 模块接收来自 Rust 解析器的 `ParsedData` 对象，其 JSON 等价格式如下：

```json
{
  "doc_id": "paper_2024_001",
  "metadata": {
    "title": "基于AI的文档审查研究",
    "page_count": 24,
    "margin_top": 2.54,
    "margin_bottom": 2.54
  },
  "sections": [
    {
      "section_id": 1,
      "type": "heading",
      "level": 1,
      "text": "1. 引言",
      "props": {
        "font-family": "SimHei",
        "font-size": "16pt",
        "line-height": "1.5"
      }
    },
    {
      "section_id": 2,
      "type": "paragraph",
      "level": 0,
      "text": "随着人工智能的发展，文档审查变得尤为重要[1]。",
      "props": {
        "font-family": "SimSun",
        "font-size": "12pt",
        "indent": "2"
      }
    }
  ],
  "references": [
    {
      "ref_id": "[1]",
      "raw_text": "[1] 张三. 人工智能导论[M]. 北京: 科学出版社, 2023.",
      "is_valid_format": true
    }
  ]
}
```

> **注意**：`Section.props` 是 `map<string, string>` 类型，所有属性值均为字符串。DRL 规则通过 `$section.getPropsMap().get("font-family")` 访问。

### 3.3 输出数据格式（AuditResponse）

```json
{
  "issues": [
    {
      "code": "ERR_FONT_001",
      "message": "一级标题必须使用黑体",
      "section_id": 1,
      "severity": "MEDIUM",
      "suggestion": "将字体修改为 SimHei（黑体）",
      "original_snippet": "1. 引言"
    }
  ],
  "score_impact": 5.0
}
```

### 3.4 错误代码规范

| 前缀 | 模块 | 示例 |
|------|------|------|
| `ERR_FONT_*` | 字体排版 | `ERR_FONT_001`（一级标题字体错误） |
| `ERR_SIZE_*` | 字号排版 | `ERR_SIZE_001`（正文字号错误） |
| `ERR_SPACING_*` | 行距/间距 | `ERR_SPACING_001`（行距不符合规范） |
| `ERR_REF_*` | 参考文献 | `ERR_REF_001`（引用格式错误） |
| `ERR_INTEGRITY_*` | 章节完整性 | `ERR_INTEGRITY_001`（缺少致谢章节） |
| `ERR_ENGINE_*` | 引擎内部错误 | `ERR_ENGINE_INIT`（规则引擎初始化失败） |

### 3.5 Severity 等级说明

| 等级 | 值 | 含义 | 扣分参考 |
|------|----|------|----------|
| `INFO` | 0 | 提示性建议，不影响合规 | 0分 |
| `LOW` | 1 | 轻微问题，建议修改 | 1-2分 |
| `MEDIUM` | 2 | 中等问题，需要修改 | 3-5分 |
| `HIGH` | 3 | 严重问题，必须修改 | 6-10分 |
| `CRITICAL` | 4 | 致命问题（如缺少必备章节） | 10分以上 |

## 六、常见问题排查

### Q1：Spring Boot 启动报 `Failed to configure a DataSource`

**原因**：`application.properties` 中数据库 URL 未配置。  
**解决**：确认 `spring.datasource.url` 已正确填写，本地用 H2，服务器用 PostgreSQL。

### Q2：Windows 控制台中文乱码

**解决方案（三选一，优先级从高到低）**：

1. 在 PowerShell 中先执行 `chcp 65001`，再运行 `mvn spring-boot:run`
2. 项目已包含 `logback-spring.xml`，会自动强制 UTF-8 输出
3. 在 `pom.xml` 的 `spring-boot-maven-plugin` 中已配置 `-Dfile.encoding=UTF-8`

### Q3：gRPC 连接被拒绝（Connection refused）

**原因**：Java 服务未启动，或端口被防火墙拦截。  
**解决**：

```bash
# 检查端口是否监听
netstat -an | grep 9191

# 服务器防火墙开放端口
sudo ufw allow 9191/tcp
sudo ufw allow 8081/tcp
```

### Q4：Drools 规则引擎初始化失败

**原因**：通常是 `kmodule.xml` 路径或 session 名称配置错误。  
**解决**：检查 `src/main/resources/META-INF/kmodule.xml` 中的 `ksession` 名称是否与代码中的 `newKieSession("formattingSession")` 一致。

### Q5：Redis 连接失败（本地开发）

**原因**：本地没有 Redis 服务，但 `application.properties` 中未禁用 Redis 自动配置。  
**解决**：确认 `application.properties` 中有以下配置：

```properties
spring.autoconfigure.exclude=org.springframework.boot.autoconfigure.data.redis.RedisAutoConfiguration,...
```
